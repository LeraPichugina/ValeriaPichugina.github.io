Проекты

Проекты "Работа с требованиями"

В рамках двух учебных кейсов проработаны этапы сбора и формализации требований к информационным системам.
Проект 1: Составлены вопросы для интервью с заказчиком по доработке системы техподдержки.
Проект 2: Сформулированы и структурированы требования для сервиса такси (бронирование поездок, оценки и отзывы, предварительный расчёт стоимости, несколько точек назначения).

Стек: Miro, Google Docs

Проект "Диаграмма BPMN: пополнение баланса телефона"

Данный проект демонстрирует бизнес-процесс пополнения мобильного счета через банковское приложение. В качестве примера рассмотрена типовая ситуация: пользователь с выключенным телефоном включает его, открывает установленное банковское приложение и пополняет баланс своего номера на 100 рублей.

Стек: BPMN 2.0, Stormbpmn

Проект "Автозаполнение реквизитов компании по ИНН с использованием DaData"

Данный проект включает в себя анализ, проектирование и моделирование пользовательского сценария автозаполнения реквизитов юридического лица в мобильном банковском приложении на основе введённого ИНН. В процессе была разработана Use Case спецификация и диаграмма последовательности UML, описывающая взаимодействие между пользователем, мобильным приложением, сервером банка и внешним API DaData.

Стек: PlantUML

Проект "Документирование API для кинотеатра"

Данный проект демонстрирует спроектированный API для сервиса онлайн-бронирования билетов в кинотеатр с бонусной программой, поддержкой просмотра сеансов, управлением бронями и формированием отчётов.

Стек: OpenAPI 3.0, SwaggerHub

Проект "ER-диаграмма: система управления заказами"

Данный проект демонстрирует спроектированную ER-диаграмму базы данных для системы заказов. Здесь учтены ключевые сущности: заказы, клиенты, товары и адреса доставки, и реализована связь многие-ко-многим между заказами и товарами. Структура таблиц спроектирована с опорой на бизнес-логику.

Стек: PostgreSQL, DBeaver
